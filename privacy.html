Política de Privacidad — TABLERO Rö

Última actualización: 19 de mayo de 2026

Qué es TABLERO Rö

TABLERO Rö es una aplicación web personal de planificación semanal que integra Google Calendar para visualizar y gestionar eventos. Es de uso exclusivamente personal y no comercial.

Autenticación con Passkey

TABLERO Rö implementa autenticación mediante passkeys (WebAuthn) para proteger el acceso a tu tablero personal. Los passkeys utilizan:

  • Autenticación biométrica (Face ID, Touch ID, Windows Hello) o PIN del dispositivo.
  • Credenciales almacenadas localmente en el hardware seguro de tu dispositivo (TPM, Secure Enclave).
  • Sin contraseñas: no se almacenan ni transmiten contraseñas.
  • Vinculación al dominio: las credenciales solo funcionan en este sitio web, protegiéndote contra phishing.

Datos biométricos: Tus datos biométricos (huella dactilar, reconocimiento facial) nunca abandonan tu dispositivo. Solo se utilizan localmente para desbloquear la clave criptográfica almacenada en tu hardware. La aplicación no tiene acceso a estos datos biométricos.

Datos que accedemos

Al conectar tu cuenta de Google, la aplicación accede a:

  • Eventos de Google Calendar (lectura): para mostrar tus eventos en el tablero semanal.
  • Creación de eventos (escritura): solo cuando tú explícitamente envías una tarea al calendario con el botón 📅.

Almacenamiento

Todos los datos se almacenan exclusivamente en tu navegador (localStorage y sessionStorage). No tenemos servidores, bases de datos ni backend. No recopilamos, transmitimos ni almacenamos datos personales en ningún servidor externo.

Datos almacenados localmente incluyen:

  • Información de usuario: nombre de usuario e ID de credencial del passkey.
  • Sesión activa: timestamp de inicio de sesión (sessionStorage, se elimina al cerrar el navegador).
  • Estados del tablero: tareas completadas, notas y configuración personal.
  • Configuración de IA: claves de API para proveedores de IA (Claude, OpenAI, Gemini) si las configuras.

Tokens de acceso

El token de Google OAuth se mantiene únicamente en la memoria del navegador durante la sesión activa. No se persiste en disco ni se envía a terceros.

Terceros

No compartimos datos con terceros. Las únicas comunicaciones de red son directamente entre tu navegador y:

  • APIs de Google Calendar (para sincronización de eventos)
  • APIs de proveedores de IA (Claude, OpenAI, Gemini) si configuras extracción de Bullet Journal

Seguridad

TABLERO Rö implementa las siguientes medidas de seguridad:

  • WebAuthn/Passkeys: estándar FIDO2 para autenticación sin contraseñas.
  • Sesiones temporales: expiración automática de sesión después de 24 horas de inactividad.
  • Sin almacenamiento de secretos: las claves de API se almacenan localmente y nunca se transmiten a nuestros servidores (porque no tenemos servidores).
  • HTTPS: toda la comunicación se realiza a través de conexiones cifradas.

Eliminación de datos

Puedes eliminar todos los datos almacenados en cualquier momento:

  • Cerrar sesión: desde ⚙️ Admin → 🔐 Seguridad · Passkey → 🚪 Cerrar sesión
  • Eliminar passkeys: desde ⚙️ Admin → 🔐 Seguridad · Passkey → 🔑 Gestionar passkeys
  • Limpiar todo: desde ⚙️ Admin → "Limpiar estado guardado", o limpiando el localStorage de tu navegador
